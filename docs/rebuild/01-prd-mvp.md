# PRD 與 MVP 範圍

## 目標

- 以新版 Laravel 重建部落格系統，保留既有 SEO 資產與文章資料。
- 提升可維護性（版本可升級）、頁面一致性（含程式碼區塊）、互動穩定性。

## 角色

- 訪客：瀏覽文章、搜尋、查看留言、按讚、分享
- 會員：登入、留言、互動
- 管理者：建立/編輯文章、審核留言、內容維護

## MVP 必做功能

1. 文章列表（分頁 URL 可索引）
2. 文章詳情（SEO metadata、程式碼區塊、分享）
3. 留言系統（新增、列表、排序、搜尋）
4. 文章搜尋（列表頁）
5. Sitemap 自動產生（至少 user_id=32 的公開文章）
6. 基本社群登入（沿用現有 social mapping）

## 可延後功能（Phase 2）

- 即時在線人數優化（WebSocket/SSE）
- 留言投票與防濫用強化
- 進階推薦、熱門關鍵字演算法
- 後台可視化儀表板

## 非功能需求

- SEO：首屏可爬、分頁可爬、canonical 正確、301 完整
- 效能：首頁 LCP < 2.5s（快取命中時）
- 穩定：上線首月 P1 錯誤率 < 0.5%
- 安全：XSS/CSRF/Rate limit 基本防護完成

## 里程碑建議

- M1（1-2 週）：資料模型 + 文章/首頁完成
- M2（1 週）：留言 + 搜尋 + SEO 線上驗證
- M3（1 週）：資料遷移 + UAT + 灰度上線
